Vamos aprender agora com API

API (Aplication Programming Interface ou Interface de Programação de Aplicações) Trata-se de um conjunto de padrões e rotinas estabelecidos e documentados por uma aplicação X, para que outras aplicações possam usar as funcionalidades desta aplicação X, sem precisar conhecer detalhes da implementação do software

Desta Forma, entendemos que as API's permitem uma interoperabilidade entre aplicações. Em outras palavras, a comunicação entre aplicações e entre os usuários.

A função deuma API é, basicamente, facilitar e simplificar o trabalho de desenvolvedores, além de oferecer um padrão para a criação de novas plataformas. Com o uso das API's, não é necessário criar códigos personalizados para cada função que um programa for execultar , o que simplica a criação de novos apliativos, softwares e plataformas em geral.


Temos alguns tipos de API: API Rest e API RestFul

REST vem do inglês "Representation State Transfer" que em português significa "Transferência de Estado Representacional"

Trata-se de uma abstração da arquitetura WEB. Resumidamente a API Rest consiste em princípios e regras que, quando seguidas, permitem a criação de um projeto com interfaces bem definidas. Desta forma, permitindo, por exemplo, que aplicações se comuniquem

RESTFUL possui os mesmos princípios da API REST... Existe uma certa confusão entre os termos "RESTE" e "RESTFUL", porém a única diferênça que existe entre esses dois termos é apenas a gramática. Em outras palavras, sistemas que ultilizam os princípios REST são chamados de RESTFUL. Ser RESTFUL é cumprirpadrões REST


REST => CONJUNTO DE PRINCÍPIOS DE ARQUITETURA

RESTFUL => API QUE SEGUE OS PRINCÍPIOS REST


Padrões REST

  • Cliente-Servidor: As aplicações existentes no servidor e no cliente devem ser separadas
  • Sem-Estado: As requisições são feitas de forma independente, ou seja, cada uam execulta uma determinada ação
  • As respostas são auto-descritivas, incluindo informação suficiente para que o cliente saiba como ultilizá-la

Portanto, quando se fala de REST API, significa ultilizar uma API para acessar aplicações back-end, de modo que essa comunicação seja feita seguindo os padões definidos pelo estilo de arquitetura REST
